# 웹 서버 게이트웨이 인터페이스 (WSGI)
초기의 공용 게이트웨이 인터페이스(CGI)는 웹 서버가 외부 프로그램을 실행하고 그 결과를 반환하도록 설계되었다. CGI는 클라이언트에게 받은 입력을 서버를 통해 처리하고 외부 프로그램으로 전달하고 프로그램은 각 클라이언트의 접근을 위해 처음부터 실행이 된다. 
이러한 방식은 시동 지연을 만들었고 개발자들은 웹 서버에 인터프리터를 두기 시작했다. 또 다른 방식으로는 별도의 프로그램 내에서 동적 언어를 실행하고 웹 서버와 통신하도록 하는 것이다. 
파이썬 웹 개발은 웹 서버 게이트웨이 인터페이스(WSGI)의 정의에서부터 시작되었다. 

# 프레임워크
프레임워크가 제공하는 최소 혹은 최대의 기능
* 라우트 - URL을 해석하여 해당 서버의 파일이나 코드를 찾음.
* 템플릿 - 서버의 데이터를 HTML 페이지에 병합함.
* 인증 및 권한 - 사용자 이름, 비밀번호, 허가를 처리함.
* 세션 - 웹 사이트에 방문하는 동안 사용자의 임시 데이터를 유지.
